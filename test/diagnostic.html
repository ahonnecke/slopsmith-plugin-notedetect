Note Detection — Layer-by-Layer Diagnostic

Open this in a tab while Slopsmith + notedetect plugin is running on the same origin. Each test isolates one layer of the detection → matching → rendering pipeline.

Test 1: Is the plugin loaded?

Checks if the notedetect plugin's globals are present in the page.




Test 2: Is detection active?


Checks _ndEnabled, audio context state, detected MIDI value.






Test 3: Are detections arriving?


Polls _ndDetectedMidi for 5 seconds. Play a note on your guitar.






Test 4: Are chart notes available?


Checks highway.getNotes() and highway.getChords().






Test 5: What's in _ndNoteResults?


Dumps the current judgment map — shows what matching has produced.






Test 6: Is _ndCheckMisses running?


Checks if the miss-check interval is active and producing miss entries.






Test 7: Force a miss and check rendering


Injects a fake MISSED_NO_DETECTION into _ndNoteResults for the first chart note
near the current time. If the draw hook works, you should see a red X on the highway.






Test 8: Timing alignment check


Shows the clock values used by matching vs the highway. Drift here = mismatched hits.






Live Event Log


Streams _ndEventLog entries as they arrive. Play notes to see matching attempts.
